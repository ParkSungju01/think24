# 홈 화면 (웹/데스크톱) 구현 계획

## 개요
- 피그마 링크: https://www.figma.com/design/Tjb8LmHOOZXQMLSL86LSJC/일상뒤집기-디자인?node-id=89-42
- fileKey: `Tjb8LmHOOZXQMLSL86LSJC`, nodeId: `89:42`
- 관련 화면: 홈 화면 (웹/데스크톱 1920×1080 기준)
- 한 줄 요약: 좌측 사이드 내비게이션 + 우측 콘텐츠(인사말/알림, 절약 금액 카드, 진행 중인 고민 카드, 월별 소비 요약 카드, 오늘의 소비 한 줄 카드)로 구성된 대시보드형 홈 화면.

### 피그마 파일 구조 관련 중요 참고사항
- 이 피그마 파일은 화면 전체를 감싸는 프레임(Frame)이 따로 없고, 배경 사각형(`89:42`, "Rectangle 6670339", 1920×1080, `#eefff0`)과 그 위에 겹쳐진 개별 텍스트/사각형/이미지 레이어(`89:43` ~ `89:125`)가 캔버스에 평면적으로 배치되어 있다. 즉 `89:42` 자체는 배경색만 가진 노드이고, 실제 화면 내용은 형제 노드들이다.
- 같은 파일 안에 모바일 버전 홈 화면(약 x=-3040 위치, 390×812 프레임 `21:346` 부근)도 별도로 존재한다. 사용자가 지정한 `node-id=89-42`는 "웹" 섹션(페이지 내 "웹" 레이블, y≈688 아래)에 속한 **데스크톱/웹 버전**이다. 이번 계획은 이 웹 버전만을 대상으로 한다 (모바일 버전은 범위 밖, 확인 필요 섹션 참고).
- Figma 변수(색상/타이포 토큰)는 `get_variable_defs` 조회 결과 없음(`{}`) — 이 파일은 변수 없이 하드코딩된 hex 값과 인라인 텍스트 스타일만 사용한다. 아래 명시된 hex 값을 그대로 사용한다.

## 전체 레이아웃 (1920×1080 기준 좌표)
- 사이드바: x=0~409, 전체 높이(1080), 배경 `#eefff0`
- 콘텐츠 영역: x=409~1920 (폭 1511), 배경 `#ffffff`
- 콘텐츠 내부 그리드 (콘텐츠 영역 기준 상대 좌표, outer padding 약 71~75px):
  - 헤더 행: 인사말 텍스트(좌) + 알림 벨 버튼(우), 상단에서 약 60~66px
  - 좌측 컬럼 (폭 836): 절약 금액 카드(위) → 진행 중인 고민 카드(아래), 두 카드 세로 스택
  - 우측 컬럼 (폭 473): 월별 소비 요약 카드, 좌측 두 카드를 합친 높이(662)와 동일한 높이로 나란히 배치
  - 하단 풀폭 카드 (폭 1375): 오늘의 소비 한 줄 카드, 좌/우 컬럼 아래 전체 폭에 걸쳐 배치
- 카드 사이 세로 간격 약 20~29px, 좌/우 컬럼 사이 간격 약 62px

  - `selectedMonth: string` — 월별 소비 요약 드롭다운 선택 값 (예: "7월"), 옵션은 해당 연도의 월만 (드롭다운 상세 인터랙션은 확인 필요)

  - 더미 데이터는 컴포넌트에 하드코딩하지 않고 `src/mocks`(예: `src/mocks/home.ts`)에 분리해 정의하고 import해서 사용한다.

## 작업 순서 제안 (developer 서브에이전트용)
1. Tailwind, react-router-dom, lucide-react 설치 및 기본 설정 (아직 미설치 상태).
2. 라우팅 스캐폴딩: `App`에 `BrowserRouter`/`Routes` 구성, `HomePage`를 `/`(또는 확정된 경로)에 연결.
3. `src/components`에 공용 레이아웃 요소(`Sidebar`, `NavItem`) 구현 — 활성 상태는 `useLocation` 기반.
4. `src/pages`에 `HomePage` 구현: 헤더(인사말+알림벨) → 콘텐츠 그리드(절약 금액 카드, 진행 중인 고민 카드, 월별 소비 요약 카드) → 하단 오늘의 소비 한 줄 카드 순으로 섹션별 컴포넌트 분리.
5. 더미 데이터 정의(사용자 확인 후 목데이터 위치 결정에 따름) 및 각 카드에 연결.
6. 아이콘/에셋 적용: 기존 `src/assets/*.svg`는 지정된 위치에, 매핑 안 된 항목은 lucide-react 아이콘으로 우선 구현 (사용자 확인 후 교체 가능하도록 컴포넌트 분리).
7. 피그마에서 추출한 색상/타이포/간격 값 그대로 Tailwind 유틸리티 또는 커스텀 클래스로 반영 (임의 변경 금지).
8. 위 "확인 필요" 항목 중 애매한 인터랙션(타이머 늘리기, 드롭다운, 알림 이동 등)은 임의 구현하지 말고 AskUserQuestion으로 확인 후 진행.
